# Feature Plans

Place major feature plans in this folder using:

- `plan/features/<feature-name>.md`

Each feature plan should include enough detail for an agent to implement independently:

- Context and background
- Feature requirements (functional and non-functional)
- User stories and acceptance criteria
- Execution plan (milestones/phases)
- Technical architecture
  - mark each architectural point as `Agreed` or `Proposed`
- Test cases
- Test methodology (manual + automated)
- Risks, dependencies, rollout, rollback, and observability notes

Recommended: copy `feature-plan-template.md` when creating a new plan.
